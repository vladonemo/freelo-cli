# 0006 — README autocheck (doc-drift guard)

**Status:** approved (autonomous run 2026-04-25-2034-readme-autocheck)
**Tier:** Yellow
**Commit type:** chore

## Problem

The published README on npmjs.com for `freelo-cli@0.3.1` still says:

> Status: **early scaffold.** The only command implemented today is `freelo --version`. Real subcommands land as their feature specs do.

Reality (R01 + R02 + R02.5 shipped):

- `auth login`, `auth logout`, `auth whoami`
- `config list`, `config get`, `config set`, `config unset`, `config profiles`, `config use`, `config resolve`
- `--introspect`, `help [cmd] --output json`

`README.md` is in `package.json` "files", so the stale copy reaches every npm consumer. Root cause: `.claude/docs/sdlc.md` Phase 6 has no README rule, so doc-writer skipped it three releases in a row.

## Proposal

Three coordinated changes in one PR:

1. Replace the "early scaffold" claim with a truthful status, and embed an **autogen Commands block** delimited by HTML comment markers. Surrounding hand-written sections (Install / Use / Develop / Process / License) stay.
2. Add `scripts/check-readme.mjs` — a Node script that renders the same block from a live `dist/freelo.js --introspect` envelope and either compares (default = CI mode) or rewrites (`--write`).
3. Add a `pnpm check:readme` gate to CI after `pnpm build`.
4. Add a Phase 6 rule to `.claude/docs/sdlc.md` so future doc-writers maintain the marker block.

### Autogen rendering format

Source: `dist/freelo.js --introspect` produces `freelo.introspect/v1` envelope where `data.commands[]` is already alphabetically sorted by `name`. Each command has `name`, `description`, `args[]`, `flags[]`, `output_schema`, `destructive`.

Rendered block (between markers, no surrounding blank lines inside the block boundary):

```markdown
<!-- BEGIN AUTOGEN COMMANDS -->
<!-- Generated by scripts/check-readme.mjs from `freelo --introspect`. Do not edit by hand. -->

### auth

- `freelo auth login` — Store credentials for a Freelo profile and verify them.
- `freelo auth logout` — Remove stored credentials for a Freelo profile.
- `freelo auth whoami` — Show the currently authenticated user.

### config

- `freelo config get <key>` — Get the current value and source of a configuration key. Read-only keys (e.g. 'apiKey') return '[redacted]'.
- `freelo config list` — List all configuration keys with current values and sources.
- `freelo config profiles` — List all configured profiles.
- `freelo config resolve` — Print the merged effective configuration with optional per-leaf source annotation.
- `freelo config set <key> <value>` — Set a configuration key. Writable keys: output, color, profile, apiBaseUrl, verbose. Read-only keys (email, apiKey) must be updated via 'freelo auth login'.
- `freelo config unset <key>` — Remove a configuration key from the user conf store. Read-only keys cannot be unset via 'config'; use 'freelo auth logout' for credentials.
- `freelo config use <profile>` — Switch the active profile. The profile must already exist (created via 'freelo auth login'). No network call is made — use auth whoami to verify credentials afterward.

<!-- END AUTOGEN COMMANDS -->
```

Rendering rules (deterministic, OS-independent):

- Group commands by **first space-separated segment** of `name` (`auth`, `config`, …). Group order = alphabetic.
- Within each group, commands are alphabetic by `name` (which `--introspect` already guarantees, but the renderer re-sorts defensively).
- Heading: `### <group>` (one blank line after).
- Each line: `` - `freelo <name>[ <arg-tokens>]` — <description.trim()> ``
  - `<arg-tokens>` is space-joined `<arg.name>` (required) or `[arg.name]` (optional). `arg.variadic` becomes `<arg.name...>` / `[arg.name...]`.
  - If `description` is empty/undefined, the em-dash and description are omitted (`- \`freelo foo\``).
- One blank line between groups; no trailing blank line inside the block.
- Line endings: LF only (`\n`). The script normalizes the README's existing CRLF to LF for comparison purposes but writes LF.
- The block's first line after `<!-- BEGIN AUTOGEN COMMANDS -->` is a fixed comment: `<!-- Generated by scripts/check-readme.mjs from \`freelo --introspect\`. Do not edit by hand. -->` followed by a blank line.

### `scripts/check-readme.mjs` CLI surface

```
node scripts/check-readme.mjs            # CI mode: exit 0 on match, 1 on drift (prints unified diff to stderr)
node scripts/check-readme.mjs --write    # rewrite README.md in place; exit 0 on success, 1 on missing markers
```

Behavior:

- Reads `dist/freelo.js` relative to repo root. If missing, prints `dist/freelo.js not found — run \`pnpm build\` first` to stderr and exits 2.
- Spawns `node dist/freelo.js --introspect` (no shell), captures stdout, parses as JSON, validates `schema === 'freelo.introspect/v1'` and `Array.isArray(data.commands)`. Validation failure → exit 2.
- Reads `README.md`. If either marker is missing, prints a clear message naming the missing marker → exit 2.
- Splices the rendered block (between markers, exclusive). Compares to current contents (after CRLF→LF normalization for both sides).
- Default: on drift, prints a unified diff (lightweight, line-by-line — no extra deps) and exits 1. On match, prints `README autogen block is up to date.` and exits 0.
- `--write`: always writes the spliced result back (LF, UTF-8, trailing newline preserved). Exits 0.

Exit-code contract:
- `0` — match (CI mode) or write succeeded.
- `1` — drift detected (CI mode only).
- `2` — environmental error (missing dist, missing markers, malformed envelope).

### CI integration

Add a single new job `check-readme` (separate from the test matrix), running on `ubuntu-latest` + Node `24`:

```yaml
check-readme:
  name: check README autogen
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: pnpm/action-setup@v4
      with: { version: 9.15.0, run_install: false }
    - uses: actions/setup-node@v4
      with: { node-version: 24, cache: pnpm }
    - run: pnpm install --frozen-lockfile
    - run: pnpm build
    - run: pnpm check:readme
```

Justification for single-row: rendering is OS- and Node-version-independent (pure string ops over JSON). The matrix already proves the build works on every OS/Node combo. Saves ~8 CI rows of duplication.

### SDLC doc update (Phase 6)

Append to `.claude/docs/sdlc.md` Phase 6 (Document):

> The published `README.md` carries an autogen Commands block between
> `<!-- BEGIN AUTOGEN COMMANDS -->` / `<!-- END AUTOGEN COMMANDS -->` markers. It is
> verified in CI by `pnpm check:readme`, which compares the committed block against
> a freshly rendered one from `dist/freelo.js --introspect`. **doc-writer must run
> `pnpm fix:readme` whenever a spec adds, removes, renames, or re-describes a public
> command**, and commit the regenerated block in the same PR. Never delete the marker
> block; if the CLI has no commands, the block stays empty between the markers.

## API surface

None. No Freelo API calls. No new envelope schemas. No exit-code changes for the CLI itself.

## Data model

The script consumes the existing `freelo.introspect/v1` envelope. No new zod schemas (script is `.mjs` and uses lightweight runtime validation — `typeof`/`Array.isArray` checks).

## Edge cases

- **Empty description**: render `- \`freelo <name>\`` with no em-dash (defensive — none today).
- **Missing markers**: exit 2 with a clear message naming which marker is missing.
- **CRLF README on Windows dev box**: normalize to LF for comparison; `--write` always emits LF.
- **`dist/freelo.js` not built**: exit 2 with hint to run `pnpm build`.
- **`--introspect` fails or is non-zero**: exit 2, surface stderr.
- **JSON parse fails or schema mismatch**: exit 2, name the field that failed validation.
- **Unsorted `commands[]` from a future introspect change**: renderer re-sorts; never trusts upstream order.
- **A command without an arg description in introspect** (existing behavior — `arg.description` is empty string): only `arg.name` is used in the rendered token, so this is a no-op.

## Non-goals

- No npm publish / release-tooling changes.
- No introspect schema/behavior changes.
- No `engines.node` change.
- No `help` command in the autogen block (intentionally excluded by introspect today).
- No README design overhaul; only the marker block is generated, surrounding prose is hand-edited.
- No telemetry, no network access in the script.

### Dependencies

No new deps. The script uses only Node built-ins (`node:fs`, `node:child_process`, `node:path`, `node:url`).

### Test strategy

`test/scripts/check-readme.test.ts` (vitest, no MSW — pure file/process I/O):

1. **Happy path (match)**: write a temp README with a correctly-rendered block, run script in CI mode against a fixture envelope (use a stub `dist/freelo.js` written to a tempdir that prints the fixture), expect exit 0.
2. **Drift (mismatch)**: write a README whose block is stale, expect exit 1 + diff in stderr.
3. **`--write`**: write a stale README, run with `--write`, expect exit 0 and the README rewritten to match.
4. **Missing markers**: README without the markers → exit 2 with marker-name in stderr.
5. **Missing `dist/freelo.js`**: point script at a tempdir without it → exit 2.
6. **Multi-group rendering**: assert that auth + config commands render under separate `### <group>` headings, alphabetically.
7. **Optional/variadic args**: feed a fixture envelope with an optional and a variadic arg, assert the rendered tokens (`[name]`, `<name...>`).

The script uses an env var `FREELO_README_REPO_ROOT` (or accepts a positional path) so the test can point it at a tempdir without polluting the real repo. **Decision**: env var — non-positional, doesn't change the documented CLI surface.

The "live" path is exercised end-to-end by CI itself (the new `check-readme` job runs against the real `dist/freelo.js`).

### Rollout order

Single PR (atomic). Commits:

1. `chore(scripts): add README autogen check` — adds the script + tests + pnpm aliases. README itself **untouched** in this commit (so `pnpm check:readme` would fail at this point — that's expected; the next commit fixes it).
2. `chore(docs): backfill README commands and wire CI gate` — runs `pnpm fix:readme` to regenerate the block, replaces "early scaffold" prose, adds the CI job, updates `.claude/docs/sdlc.md`, adds the changeset.

After commit 2, the full gate runs clean: `pnpm typecheck && pnpm lint && pnpm test && pnpm build && pnpm check:readme`.

### Decision log

- **D1** — Use HTML-comment markers (`<!-- BEGIN AUTOGEN COMMANDS -->` / `<!-- END AUTOGEN COMMANDS -->`). Stable, invisible in rendered Markdown, well-known pattern.
- **D2** — Group by first segment, sort alphabetically. Deterministic; matches today's flat introspect order.
- **D3** — Script consumes existing `dist/freelo.js`; doesn't run `pnpm build` itself. Composes with the CI build step; keeps the script fast and side-effect-free.
- **D4** — Single CI job (`ubuntu-latest`, Node 24). Output is OS- and Node-independent; matrix would burn 8 redundant rows.
- **D5** — `freelo-cli: patch` changeset. README is in `files`; npm consumers see new content.
- **D6** — Test uses `FREELO_README_REPO_ROOT` env var to point the script at a tempdir. Avoids changing the documented CLI surface.
